This result is based on the window.screen.width and window.screen.height properties provided by JavaScript. These values are multiplied by the window.devicePixelRatio to get the above result. This compensates for OS scaling.

Many other online resolution checking websites fail to do this, and they will therefore give incorrect results when OS scaling is enabled (such as with Apple computers with a "Retina" display, or high-resolution displays on Windows).

In devices with arbitrary scaling ratios (particularly mobile devices), some unavoidable error is introduced because window.screen.width and height are rounded to integer values. Therefore, an uncertainty range (i.e. "±1 px") is provided in that situation, which represents the maximum possible deviation from the true value.

However, the above result may still be incorrect due to various other circumstances, such as using browzer zoom. OS scaling changes the values of window.screen.width and window.devicePixelRatio inversely, but browser zoom changes the devicePixelRatio without changing the values of window.screen.width or height. This means that the calculation to obtain the original screen resolution is different depending on whether browser zoom or OS scaling is in effect.

Since both OS scaling and browser zoom are combined into the device pixel ratio value, so there is no way to tell which one is being used. Therefore, this calculator assumes the OS scaling is in effect, since this is a more common situation.
